## вңЁ Features

- рҹ”Қ **Real-time web search** via Tavily on every query (basic or advanced depth, configurable by mode)
- вҡЎ **Streaming AI responses** over Server-Sent Events вҖ” answer chunks rendered progressively as named `delta` events
- рҹ§  **Model selection** вҖ” pick between `gpt-5.5`, `gpt-5.4`, and `gpt-5.4-mini`
- рҹ”Җ **Fast / Thinking modes** вҖ” Fast = basic search (10 results); Thinking = advanced search (20 results)
- рҹ’¬ **Persistent conversations** вҖ” full message history stored in PostgreSQL via Drizzle ORM, with sources persisted alongside each assistant message
- рҹҺҷпёҸ **Voice transcription** вҖ” click the mic in the ChatBar, speak, and your words stream into the textarea via OpenAI `gpt-4o-mini-transcribe`
- рҹҺЁ **Image Studio** вҖ” generate images from a prompt with `gpt-image-1` / `gpt-image-2`; pick from 15 styles, 4 aspect sizes, quality, and output-format controls
- рҹ–јпёҸ **Image editing** вҖ” upload one or more reference images and transform them with a prompt
- рҹ“Ў **Streaming image preview** вҖ” partial base64 frames render progressively while the image is generated
- рҹ—ӮпёҸ **Media Library** вҖ” browse, download, re-edit, and **delete** generated images (30-day expiry), backed by Supabase Storage
- вңҸпёҸ **Rename & delete conversations** вҖ” hover any item in the history sidebar for inline rename (Enter saves, Esc cancels) and inline delete confirm
- рҹӘҹ **Sources tab** вҖ” top-level Chat / Sources switcher in each conversation; aggregates every unique source cited in the thread with favicon + hostname
- рҹ”— **Inline sources panel** вҖ” web result cards surface alongside each individual answer
- рҹӨ– **Auto-generated follow-up questions** suggested after every answer
- рҹ‘Ө **User menu with hover popover** вҖ” avatar in the sidebar footer; popover shows name, email, OAuth provider, and remaining credits
- рҹҶ• **Centered `/chat/new` hero** вҖ” fresh-chat lives inside the chat layout (sidebar visible), not on a separate page
- рҹ“ҡ **Searchable history tab** in the sidebar with relative timestamps
- рҹҡҖ **Agent Mode tab** вҖ” placeholder with animated "Coming Soon" badge
- рҹҺЁ **Microinteractions** вҖ” spring-animated mode toggle, model dropdown, staggered list animations via Framer Motion
- рҹ—ғпёҸ **Semantic search caching** via Pinecone (skips Tavily on high-similarity hits)
- рҹ—„пёҸ **Redis response caching** on conversations & messages (1-hour TTL)
- рҹ”җ **GitHub & Google OAuth** via Supabase вҖ” with PKCE flow, auto token refresh, and `?next=` post-login redirect preservation
- рҹҡҰ **Per-user rate limiting** вҖ” 20 requests per minute on chat and transcription endpoints

## рҹҺЁ Image Studio

Beyond chat, Bunplexity ships a full **Image Studio** (`/studio`) and **Media Library** (`/library`), reachable from the sidebar. The Studio supports two modes:

- **Generate** вҖ” create an image from a text prompt.
- **Edit** вҖ” upload one or more reference images and transform them with a prompt (multipart upload, up to 5 images, 10 MB each).

Generation streams a live base64 preview while the model works, then uploads the final image to Supabase Storage and returns a durable public URL. Every image is owned by the user, listed in the Library, and **expires after 30 days**. Both the Studio "this session" gallery and the Library let you download, re-edit, and delete.

| Option | Values |
|--------|--------|
| **Style (15)** | Realistic *(default)*, Cinematic, Anime, 3D Render, Watercolor, Oil Painting, Pixel Art, Comic Book, Minimalist, Cyberpunk, Fantasy, Vintage, Sketch, Cartoon, Studio Photo |
| **Size** | `auto` *(default)*, `1024x1024` (Square), `1536x1024` (Landscape), `1024x1536` (Portrait) |
| **Quality** | `auto` *(default)*, `high`, `medium`, `low` |
| **Model** | `gpt-image-1` *(default)*, `gpt-image-2` |
| **Output format** | `jpeg` *(default)*, `png`, `webp` |

## рҹҡҖ Getting Started

### Prerequisites

- [Bun](https://bun.sh) вүҘ 1.0
- [Docker](https://docker.com) (for Redis)
- API keys: [Supabase](https://supabase.com) В· [OpenAI](https://platform.openai.com) В· [Tavily](https://tavily.com) В· [Pinecone](https://pinecone.io)
- A **1024-dimension** Pinecone index (the backend calls `text-embedding-3-small` with `dimensions: 1024`)
- A **public** Supabase Storage bucket named **`Bunplexity-Images`** (the Image Studio serves images via public URLs, so the bucket must be public for `<img>` tags to load them)

---

### Setup

```bash
git clone https://github.com/your-username/bunplexity.git
```

**1. Start Redis**

```bash
docker run -d -p 6379:6379 redis:7-alpine
```

**2. Backend**

```bash
cd backend
cp .env.example .env       # fill in all API keys
bun install
bunx drizzle-kit push      # sync schema (sources jsonb + images table)
bun run dev                # http://localhost:3001
```

> **Storage bucket:** in the Supabase dashboard вҶ’ Storage, create a **public** bucket named `Bunplexity-Images` before generating images.

**3. Frontend**

```bash
cd frontend
bun install
bun run dev                # http://localhost:3000
```

> **Without Redis:** the backend starts fine and silently skips caching вҖ” all features still work without the 1-hour response cache.

## рҹ”„ How It Works

1. **User submits a query** with a selected model and mode вҶ’ frontend POSTs `{ query, conversationId, mode, model }` with a Supabase Bearer token. For `/chat/new`, the frontend mints a fresh UUID before the first send.
2. **Auth & rate limit** вҶ’ backend validates the JWT via Supabase, enforces 20 req/min per user.
3. **Embedding** вҶ’ query is embedded via `text-embedding-3-small` (1024 dims) in parallel with conversation setup.
4. **Semantic cache check** вҶ’ embedding compared against **Pinecone**; a hit returns cached web results, skipping Tavily.
5. **Web search** вҶ’ if no cache hit, query sent to **Tavily** (10 results in Fast, 20 in Thinking).
6. **LLM inference** вҶ’ search results + query injected into a structured prompt, streamed through **OpenAI Responses API** using the chosen model.
7. **SSE to browser** вҶ’ `event: delta` chunks render progressively, followed by `event: sources` and `event: done`. Real upstream errors propagate as `event: error` with sanitized messages.
8. **Persistence** вҶ’ assistant message + sources written to **PostgreSQL** (`messages.sources` jsonb); conversation lists invalidated in **Redis**.
9. **Voice input (optional)** вҶ’ user clicks the ChatBar mic. `MediaRecorder` captures audio, posts to `/transcript`, and the backend streams transcription deltas through `gpt-4o-mini-transcribe` straight into the textarea.
10. **Image generation (optional)** вҶ’ in the Studio, prompt + options post to `/image/generate` (or `/image/edit` with reference uploads). The **OpenAI Images API** streams `partial` base64 previews over SSE; the final image is uploaded to the `Bunplexity-Images` bucket with the correct Content-Type/extension, a row is written to **PostgreSQL**, and a `saved` frame returns the durable public URL. The **Library** (`/image/history`) lists every non-expired image for download, re-edit, or delete.

## рҹ”җ Auth Flow

- Browser uses Supabase's **PKCE** flow with `detectSessionInUrl` and `autoRefreshToken` enabled.
- A single `useAuth` hook subscribes to `onAuthStateChange`, so the access token in memory is always fresh вҖ” long chat sessions keep working past the 1-hour token expiry, and a sign-out in another tab is reflected within ~1 second.
- Protected routes hitting an unauthenticated state push the user to `/login?next=<original-path>`. After sign-in, OAuth `redirectTo` honors `next` and lands them on the originally requested URL.
- `/login` itself bounces already-signed-in users to their `next` destination (or `/chat/new`).
