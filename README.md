## вңЁ Features

- рҹ”Қ **Real-time web search** via Tavily (10 results, advanced depth) on every query
- вҡЎ **Streaming AI responses** using Server-Sent Events with NVIDIA NIM MiniMax-M2.7
- рҹ’¬ **Persistent conversations** вҖ” full message history stored in PostgreSQL
- рҹ”— **Clickable sources panel** with titles pulled from Tavily web results
- рҹӨ– **Auto-generated follow-up questions** suggested after every answer
- рҹ§  **Semantic search caching** via Pinecone (skips LLM call on вүҘ0.88 similarity)
- рҹ—„пёҸ **Redis response caching** on conversations & messages (1-hour TTL)
- рҹ”җ **GitHub & Google OAuth** вҖ” zero-password sign-in via Supabase Auth
- рҹҡҰ **Per-user rate limiting** вҖ” 20 requests per minute on the chat endpoint
- рҹҗі **Docker Compose ready** вҖ” spins up backend + Redis with a single command

## рҹҡҖ Getting Started

### Prerequisites

- [Bun](https://bun.sh) вүҘ 1.0
- [Docker](https://docker.com) & Docker Compose (for Redis)
- API keys & accounts for: [Supabase](https://supabase.com) В· [NVIDIA NIM](https://build.nvidia.com) В· [Tavily](https://tavily.com) В· [Pinecone](https://pinecone.io) В· [OpenAI](https://platform.openai.com)

---

### Option A вҖ” Docker (backend + Redis, recommended)

```bash
git clone https://github.com/your-username/bunplexity.git
cd bunplexity/backend

cp .env.example .env
# Fill in all API keys in .env

docker compose up -d
# Backend вҶ’ http://localhost:3001
# Redis   вҶ’ localhost:6379
```

Then start the frontend:

```bash
cd ../frontend
bun install
bun run dev
# Frontend вҶ’ http://localhost:3000
```

## рҹ”„ How It Works

1. **User submits a query** вҶ’ frontend POSTs `{ query, conversationId }` with a Supabase Bearer token
2. **Auth & rate limit** вҶ’ backend validates the JWT via Supabase, enforces 20 req/min per user
3. **Web search** вҶ’ query sent to **Tavily** вҖ” returns 10 live web results
4. **Semantic cache check** вҶ’ query embedding compared against **Pinecone**; a hit at вүҘ0.88 cosine similarity short-circuits the LLM call entirely
5. **LLM inference** вҶ’ search results + query injected into a structured prompt, streamed through **NVIDIA NIM** (MiniMax-M2.7)
6. **SSE to browser** вҶ’ answer chunks, sources, and follow-up questions arrive as named events; frontend renders progressively
7. **Persistence** вҶ’ complete assistant message written to **PostgreSQL**; conversation lists refreshed in **Redis**
